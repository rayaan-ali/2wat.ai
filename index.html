# wat.ai CLUB WEBSITE — Starter This textdoc contains a one-file starter website (`index.html`) you can edit and deploy as the **official wat.ai club site**. It also includes **example serverless function** code (Netlify / Vercel) you can use to power a real chatbot securely (API key hidden in server env). --- ## What I included 1. `index.html` — a complete, responsive, Tailwind-powered single-file website for a student AI club (Wat.AI). It contains: Hero, About, Projects, Get Involved, Contact, and a **chatbot widget**. The chatbot runs in two modes: - **Demo mode (default)** — a rules-based local bot (great for demos; no API keys). - **Real API mode** — toggled via a variable; it calls a serverless proxy endpoint (examples included) which forwards requests to an AI provider (OpenAI). *Do not put API keys into client-side code.* 2. `netlify/functions/openai-chat.js` — example Netlify function (Node) that accepts `{ message }` and calls the OpenAI Chat API. Set `OPENAI_API_KEY` in Netlify env vars before deploying. 3. `api/openai-chat.js` — example Vercel serverless function for the same purpose. 4. The `index.html` is a single file — everything lives there so it's easy to edit. 2. **Test locally**: - Quick: just double-click `index.html` to open in your browser. That works for static content. - Better (recommended): run a simple server so fetch behaves the same as deployed: in the folder with `index.html` open a terminal and run: - `python -m http.server 8000` (if you have Python) - then open `http://localhost:8000`. 3. **Demo chatbot (no API)**: the site defaults to a rules-based demo bot. Good for showing the club a working site without needing any keys. 4. **If you want a real AI chatbot** (optional): - Sign up for an AI provider (OpenAI is common) and get an API key. - Deploy the serverless function example (Netlify or Vercel) from the code blocks above. Put each function file in the correct folder in your repo (Netlify: `netlify/functions/openai-chat.js`; Vercel: `api/openai-chat.js`). - Add environment variable `OPENAI_API_KEY` with your key on the hosting provider (Netlify / Vercel settings). - After deploy, set `API_MODE = true` in `index.html` and update `API_ENDPOINT` to the function path (`/.netlify/functions/openai-chat` for Netlify or `/api/openai-chat` for Vercel). - **Important**: Never paste the API key directly into `index.html`. 5. **Create a GitHub repo & push** (simple commands): - `git init` - `git add .` - `git commit -m "Initial wat.ai site"` - Create an empty GitHub repo (for example `wat-ai-site`) and follow the instructions GitHub gives to push (or use: `git remote add origin ` then `git push -u origin main`). 6. **Deploy**: - **Netlify** (easy, supports serverless functions out of the box): Connect your GitHub repo, set build command to blank (static), set publish directory to `/` (root). Add `OPENAI_API_KEY` in Site settings > Environment. - **Vercel** (also easy): Import repo on Vercel, set env var in Project Settings, deploy. - **GitHub Pages**: Works for static-only site (demo chatbot only). GitHub Pages **cannot** securely host serverless functions — use Netlify/Vercel if you want real AI. 7. **Share with the club**: once deployed, you'll have a URL (e.g., `https://yourname.github.io/repo` or `https://your-site.netlify.app`). Send it to the Wat.AI leads with a short note offering to hand it over or to keep editing. 8. **Assets & content to ask the club for** (when you hand it over): - Official club logo (SVG/PNG) - Short mission statement (1–2 sentences) - Current projects (short descriptions) - Meeting schedule & location - Discord invite link / GitHub org links - Email address they want publicly listed 9. **Security & privacy**: Do NOT store API keys in the client.
